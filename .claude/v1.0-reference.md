## v1.0 JSON Metadata Rules

**Structure**
- Every JSON MUST start with `{"datasets": [{"id": ...}]}` wrapper — never unwrapped
- If a field has no confirmed value, SKIP it — do not add placeholders or invented values
- Do not create fields outside the schema/template — fields are fixed
- Only use values sourced from: data review, report, data files, or codelists
- For `details` field: content from data review/report NOT already in `description`

**Entity fields** (publisher, contact_point, creator, attributions.entity)
- `name` always REQUIRED
- At least one of `email` or `url` REQUIRED (schema `anyOf`)
- Do not invent email addresses

**Source fields** (lineage.sources)
- Schema field is `name`, NOT `title` — v0.3 uses `title`, v1.0 uses `name`
- Optional enrichment: `type` (dataset/model), `risk_data_type`, `used_in`, `license`

## Key Differences from v0.3

| Feature | v0.3 | v1.0 |
|---------|------|------|
| `publisher/creator/contact_point` | Inside `attributions` array | Separate required top-level fields |
| `project` | Object `{name, url}` | Simple string |
| `sources` | Top-level `sources[]` | Inside `lineage.sources[]` |
| `license` | String codelist code | IRI string (URL) |
| `data_format` (resources) | Closed codelist | Replaced by `media_type` + `format` |
| `hazard_type` codelist | 11 types | +`convective_storm`, `dust_sand_storm`, `erosion`, `pest_infestation`, `sea_level_rise` |
| `process_type` codelist | No lightning | +`lightning`, `thunderstorm`, `hail`, `glacial_lake_outburst`, `coastal_erosion`, `soil_erosion`, `subsidence_uplift` |
| `occurrence` types | `probabilistic` only | +`empirical`, `deterministic` with `index_criteria` |
| `climate` on resources | Not available | `{model, scenario, percentile}` |
| `baseline_period` | Not available | Period object on resources |
| `spatial_aggregation` | Not available | Free text on resources |
| `loss.losses[].disaster_identifiers` | Not available | Optional array of Classification objects (GLIDE/EM-DAT event IDs) |

## Codelist Rules

**Closed** (must match exactly): `hazard_type`, `process_type`, `climate_scenario`, `analysis_type`, `frequency_distribution`, `risk_data_type`, `spatial_scale`, `data_calculation_type`, `source_type`, `function_approach`, `impact_type`, `loss_type`, `metric_dimension`, `relationship_type`, `exposure_category`, `seasonality`, `unit_currency`
→ Currently 18 closed CSVs at `rdl-standard/schema/codelists/closed/`

**Open** (standard preferred, custom allowed): `imt_*.csv`, `IMT.csv`, `unit_*.csv`, `roles.csv`, `license.csv`, `media_type.csv`, `location_gazetteers.csv`, `classification_scheme.csv`
→ Currently 34 open CSVs at `rdl-standard/schema/codelists/open/`

**IMT codes**: Format is `metric:unit` (e.g., `AirTemp:C`, `PGWS:m/s`, `pptn24:mm`, `HD:-`). Master `IMT.csv` has `Hazard` column linking codes to types, plus `universal` entries. Per-type files `imt_[type].csv` have type-specific codes. Validator checks both combined. **No `imt_sea_level_rise.csv`** — use IMT.csv universal entries for sea_level_rise.

**earthquake process inconsistency**: `process_type.csv` (closed codelist) has `rupture`. Schema conditional has `primary_rupture`/`secondary_rupture` — this is a schema bug. Use `rupture` (the CSV value) for earthquake events.

**Climate scenario**: Single string enum — cannot hold multiple scenarios. For multi-scenario resources: omit `scenario`, use `model` and `description`. Only set when resource covers exactly one scenario.

## Cross-Field Rules (8 rules)

1. **type → process**: Process must match hazard type (full mappings in template)
2. **type → IMT**: IMT codelist switches per type, all open
3. **quantity_kind → unit**: Unit codelist per quantity_kind. Currency is CLOSED (ISO 4217).
   Use `normalise_unit()` from `src/codelists.py` before writing any unit value.
   Common confirmed codes: `square_metre` (m2), `hectare` (ha), `metre` (m), `kilometre` (km)
4. **scale → countries**: global=none, regional=min 2, national/sub-national/urban=min 1
5. **analysis_type → occurrence**: probabilistic/empirical/deterministic must match. Do NOT mix within one event_set — split into separate event_sets
6. **Entity contact**: name + (email or url) required
7. **risk_data_type → sections**: hazard/exposure/vulnerability/loss sections match risk_data_type
8. **climate.scenario → baseline_period**: scenario present → baseline_period expected

## Event Set Design

- Split by analysis type: deterministic (means), probabilistic (return periods), empirical (observations) in separate event_sets
- `occurrence_range`: only for `analysis_type: "probabilistic"` — schema guidance
- `event_count`: only when events array is NOT populated
- Mean values: use deterministic event_set with `index_criteria`, or probabilistic with `return_period: 1`

## Common Pitfalls

| Issue | Fix |
|-------|-----|
| `sources[].title` instead of `name` | v1.0 uses `name`, not `title` |
| `climate.scenario` with multiple RCPs | Omit scenario, describe in `description` |
| Custom IMT like `deg_C` instead of `AirTemp:C` | Use standard codelist codes from IMT.csv |
| Mixed occurrence types in one event_set | Split into separate event_sets per analysis_type |
| `occurrence_range` on deterministic sets | Remove — only for probabilistic |
| Missing `{"datasets": [...]}` wrapper | Always wrap |

## Post-conversion checklist

**Run after EVERY v0.3 -> v1.0 conversion before considering it done.**

### Step 1 - Run the mechanical enrichment script

```bash
C:/Users/benny/miniforge3/envs/to-rdls/python.exe scripts/post_convert_enrich.py "output/<collection>/**/*.json"
```

This applies automatically:
- `unit=count` on every metric where `quantity_kind=count` and `unit` is absent
- GED4ALL URI fix: broken gemproducts URL (404) -> wiki URL
- `scheme="Custom"` removed (not a valid `classification_scheme.csv` code)
- `"Commercial"` and TODO strings removed from `lineage.sources[].license`
- `intensity_measure="wd:m"` on flood/coastal_flood losses with TODO IM
- `scheme="GED4ALL"` restored for known GED4ALL exposure item IDs

### Step 2 - Fix asset_type manually (per dataset)

For every exposure item the script flags as needing attention, set:
- `asset_type.id`: meaningful per-item identifier - e.g. `"buildings"`, `"population"`, `"roads"` - NEVER the scheme name (`"GED4ALL"`, `"Custom"`)
- `asset_type.title`: human-readable label (e.g. `"Building taxonomy"`, `"Population taxonomy"`)
- `asset_type.description`: data source + classification context sentence
- `asset_type.scheme`: `"GED4ALL"` if exposure fits any of the 10 GED4ALL categories; absent otherwise
- `asset_type.uri`: `https://wiki.openstreetmap.org/wiki/GED4ALL` for GED4ALL items; absent otherwise

Follow the pattern in `temp/enrich_wbgufra_asset_type.py` for dataset-specific enrichment scripts.

#### GED4ALL 10 categories (scheme="GED4ALL" applies)
1. Complete Buildings  2. Simplified Buildings  3. Road Network  4. Railway Network
5. Bridges  6. Pipelines and Storage Tanks  7. Power Grids  8. Energy Generation
9. Crops, Livestock and Forestry  10. Socio-Economic (population, GDP, economic indicators)

#### No scheme (absent) for
- `land_cover`, `exp_metrics_1` - general land cover not specifically in cat 9
- `exposure_mining` - no mining category in GED4ALL or any other open codelist scheme
- `exposure_landuse` - land use zoning, no matching scheme
- Any item using a **custom taxonomy** (Deltares FIAT typology, project-specific classification)

### Step 3 - Check media_type vs format

```bash
grep -r '"format"' output/<collection>/  # catch any stragglers
```

Rule: check `rdl-standard/schema/codelists/open/media_type.csv` first.
- If format has an IANA entry -> use `media_type` field, delete `format`
- If no entry -> use `format` (free text), no `media_type`
- NEVER use both together

Common mappings:
| Format | Field | Value |
|--------|-------|-------|
| File Geodatabase (gdb) | media_type | `application/x-filegdb` |
| GeoTIFF | media_type | `image/tiff;application=geotiff` |
| Shapefile | media_type | `application/x-shapefile` |
| GeoPackage | media_type | `application/geopackage+sqlite3` |
| GeoJSON | media_type | `application/geo+json` |
| CSV | media_type | `text/csv` |

```
<original>.json   ->  <original>_v03.json   (backup of v0.3 source)
<converted>.json  ->  <original>.json       (v1.0 takes original name, no _v10 suffix)
```

Exclude backups from all scripts: `if "_v03" not in f.name`
